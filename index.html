LEVEL 1 — HTML

Pengenalan HTML

HTML (HyperText Markup Language) adalah bahasa standar untuk membuat halaman web. HTML bukan bahasa pemrograman, melainkan bahasa markup yang mendefinisikan struktur konten.

Struktur Dasar HTML:
<!DOCTYPE html>
<html lang="id">
  <head>
    <meta charset="UTF-8">
    <title>Halaman Pertamaku</title>
  </head>
  <body>
    <h1>Halo Dunia!</h1>
    <p>Ini paragraf pertamaku.</p>
  </body>
</html>

• <!DOCTYPE html> — Memberitahu browser ini adalah HTML5
• <html> — Tag pembuka dokumen HTML
• <head> — Berisi informasi meta, tidak ditampilkan
• <body> — Berisi konten yang ditampilkan

LEVEL 38 — REACT

useState

useState untuk menyimpan dan update data dalam komponen.

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Hitungan: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

Penting:
• JANGAN ubah state langsung: count = 5 ❌
• Selalu pakai setter: setCount(5) ✅
• State update = re-render komponen
• State dengan object: setUser({...user, nama: "Baru"})

LEVEL 39 — REACT

useEffect

useEffect untuk side effects (fetch data, subscribe, timer dll).

import { useEffect, useState } from "react";

function Component() {
  const [data, setData] = useState(null);

  // Jalankan setiap render
  useEffect(() => { console.log("render!"); });

  // Jalankan sekali saat mount
  useEffect(() => {
    fetch("/api/data")
      .then(res => res.json())
      .then(d => setData(d));
  }, []);

  // Jalankan saat 'id' berubah
  useEffect(() => {
    fetchData(id);
  }, [id]);

  // Cleanup (saat unmount)
  useEffect(() => {
    const timer = setInterval(() => {}, 1000);
    return () => clearInterval(timer); // cleanup
  }, []);
}

Express.js

LEVEL 48 — BACKEND

REST API

REST API = cara komunikasi client-server via HTTP.

HTTP Methods:
GET    /users        → ambil semua user
GET    /users/1      → ambil user id=1
POST   /users        → buat user baru
PUT    /users/1      → update user id=1 (seluruhnya)
PATCH  /users/1      → update sebagian
DELETE /users/1      → hapus user id=1

Status Codes:
200 OK              → berhasil
201 Created         → berhasil dibuat
400 Bad Request     → request salah
401 Unauthorized    → perlu login
403 Forbidden       → tidak punya akses
404 Not Found       → tidak ditemukan
500 Internal Error  → error server

Contoh Response JSON:
{
  "status": "success",
  "data": { "id": 1, "nama": "Hasbhi" },
  "message": "User ditemukan"
}

LEVEL 50 — BACKEND

Authentication

Autentikasi = memverifikasi identitas pengguna.

JWT (JSON Web Token):
npm install jsonwebtoken bcryptjs

// Hash password saat register
const bcrypt = require("bcryptjs");
const hash = await bcrypt.hash(password, 10);

// Verifikasi saat login
const valid = await bcrypt.compare(inputPassword, hash);

// Buat JWT token
const jwt = require("jsonwebtoken");
const token = jwt.sign(
  { id: user.id, email: user.email },
  process.env.JWT_SECRET,
  { expiresIn: "7d" }
);

// Middleware verifikasi token
function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.split(" ")[1];
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch {
    res.status(401).json({ error: "Token tidak valid" });
  }
}

LEVEL 57 — ADVANCED

Performance

Optimasi performa website agar loading cepat.

Core Web Vitals:
• LCP (Largest Contentful Paint) < 2.5s
• FID (First Input Delay) < 100ms
• CLS (Cumulative Layout Shift) < 0.1

Optimasi Gambar:
• Gunakan format WebP/AVIF
• Compress gambar (TinyPNG, Squoosh)
• Lazy loading: <img loading="lazy">
• Ukuran yang tepat (jangan 4K untuk thumbnail)

JavaScript:
• Minify & bundle (Vite/Webpack otomatis)
• Code splitting (dynamic import)
• Tree shaking (hapus kode tidak terpakai)

CSS:
• Minify CSS
• Hapus CSS tidak terpakai (PurgeCSS)
• Critical CSS inline

Caching:
• Browser cache (Cache-Control header)
• CDN untuk asset statis
• Service Worker (PWA)

Tools:
• Lighthouse (Chrome DevTools)
• PageSpeed Insights
• WebPageTest
